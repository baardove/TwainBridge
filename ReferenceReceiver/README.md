# TwainBridge reference receiver

This dependency-free Python fixture implements the PRD upload contract and deliberate failure modes. It is for local integration testing only.

TwainBridge requires HTTPS. Start the receiver with a certificate trusted by the test Mac:

```sh
python3 server.py --cert localhost.pem --key localhost-key.pem
```

Set a destination URL to `https://localhost:8443/upload`. A certificate generated by `mkcert localhost 127.0.0.1 ::1` is convenient because `mkcert -install` adds its local CA to macOS trust. Do not disable TLS validation in TwainBridge.

The query parameter `mode` selects behavior:

| Mode | Behavior |
|---|---|
| `success` | Standard JSON and per-document batch results |
| `partial` | Last document reports application failure |
| `retry` | Returns 503 before succeeding; use `failures` and `retry_after` |
| `reject` | Returns an application rejection |
| `malformed` | Returns invalid JSON |
| `oversized` | Returns a body larger than the default response limit |
| `empty` | Returns an empty 204 response |
| `status-only` | Returns a body-free 204 for status-only profiles |
| `slow` | Delays the response; use `seconds` |

`GET /state` reports only test-operation metadata and SHA-256 file hashes, so retries and idempotency can be inspected without retaining uploaded payloads. `GET /health` is a readiness check.

For a receiver-only smoke test, `--http` is available. The app intentionally refuses that endpoint because production destination URLs must use HTTPS.

Run the repeatable receiver contract test with:

```sh
Scripts/test-reference-receiver.sh
```

It validates health, normal success, a 503/503/200 idempotent retry sequence, partial document results, malformed responses, application rejection, file hashes, and metadata-only state. The complete project preflight runs this test automatically.
