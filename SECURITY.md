# Security policy

## Reporting a vulnerability

Please do not publish suspected vulnerabilities, credentials, or sensitive documents in a public GitHub issue.

Report security concerns privately to **Bård Ove Myhr** at [bard.myhr@gmail.com](mailto:bard.myhr@gmail.com). Include the affected version, reproduction steps, and impact when possible. Do not attach real scanned documents or production credentials; use sanitized test data.

## Repository hygiene

Local receiver uploads, build products, release configuration, environment files, signing keys, certificates, and provisioning profiles are excluded from source control. The committed release configuration is a placeholder template only.
